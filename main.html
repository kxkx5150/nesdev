Nerdy Nights week 1: Number Systems and Core Programming Ideas

数字システム
10進法
10進法は10を基本としている。各桁は0〜9である。各桁の位置は10の累乗です。左側の各桁は、前の桁の10倍である。例えば、10の右隣に0を置くと100となり、10倍となる。右から0を取り除くと1になり、10倍少なくなります。
100の位 10の位 1の位 0 0 1 = 001 0 1 = 010 1 0 0 = 100

数字の値を得るには、各桁にその位を掛けて、全部を足します。
100 の位 10 の位 1 の位 3 8 0 = 3*100 + 8*10 + 0*1 = 380 0 4 1 = 0*100 + 4*10 + 1*1 = 41

2進法
コンピュータの演算はすべて2進数で行われる。これは、コンピュータの最下層がスイッチであり、オン/オフ、1/0であるためである。

2進法は、各桁が0〜1であることと、位取りが10の代わりに2の累乗であることを除けば、同じように機能する。数字の右側に0を入れると2倍になる。0を取り除くと、2倍小さくなります。
8の位 4の位 2の位 1の位 0 1 0 0 = 0*8 + 1*4 + 0*2 + 0*1 = 4 1 1 1 = 1*8 + 1*4 + 1*2 + 1*1 = 15

ファミコンは8ビットシステムです。つまり、ファミコンが扱う2進数は8桁の長さです。8ビットは1バイトです。いくつかの例を挙げます。
 2進数 00000000 = 0 00001111 = 15 00010000 = 16 10101010 = 170 11111111 = 255

最終的には、2進数を読むのが速くなるか、少なくともパターンを認識できるようになります。1バイトは0〜255の範囲しかないことがわかります。これより大きな数値は2バイト以上使用しなければなりません。また、負の数は存在しません。詳しくは後述します。


16進法（Hexadecimal
16進数は16進数なので、各桁は0から15で、各桁の位置は16の累乗となります。問題は、10以上の数字が2桁必要なことです。これを解決するために、Aから始まる数字の代わりに文字が使われる。
10進数 16進数 0 = 0 1 = 1 9 = 9 10 = A 11 = B 12 = C 13 = D 14 = E 15 = F

10進数と16進数の場合と同様に、桁の位置はそれぞれ16の累乗となります。
 16の位 1の位 6 A = 6*16 + A(10)*1 = 106 1 0 = 1*16 + 0*1 = 16

16進数は、2進数よりも高速に書けるため、主に使われている。8桁の2進数は2桁の16進数になります。
バイナリ 01101010 を半分に分割 / ╱ 0110 1010 を 16 進数に変換 / ╱ 6A に戻す

 01101010 = 6A


その他の例： 
2進数 16進数 00000000 = 00 = 0 00001111 = 0F = 15 00010000 = 10 = 16 10101010 = AA = 170 111111 = FF = 255

Windowsに内蔵されている電卓を開き、科学モードに切り替えると、簡単に変換できます。 基数（Hex、Dec、Bin）を選択し、数字を入力し、別の基数に切り替わります。

数字を入力するときに、どの基数を使用しているかがわかるように、追加の文字が表示されます。2進数は、通常、%00001111のように、先頭に%が付いています。その他、2進数には00001111bのようにbを後置し、16進数には2Ahのようにhを後置する慣例もあります。

ファミコンのアドレスバスは16ビットなので、2^16バイトのメモリにアクセスすることができます。16桁の2進数は4桁の16進数になるので、典型的なファミコンのアドレスは$8000, $FFFF, $4017 のようなものです。


プログラミングの基本概念
すべてのプログラミング言語には、3つの基本概念があります。命令、変数、制御フローである。この3つのうち1つでも欠けていれば、それはもう本当のプログラミング言語とは言えません。例えば、HTMLには制御フローがないので、プログラミング言語とは言えません。

命令
命令とは、プロセッサが実行する最小のコマンドのことである。命令は一度に一つずつ、次々と実行される。ファミコンのプロセッサには56個の命令しかない。そのうち10個程度は常に使用され、少なくとも10個は完全に無視される。例えば、足し算、数値のロード、変数とゼロの比較などである。

変数
変数とは、変更可能なデータを格納する場所である。例えば、画面上のマリオの垂直位置などである。ゲーム中にいつでも変更することができる。ソースコードの変数はすべて自分で名前をつけるので、MarioHorizPositionのような名前になる。

制御の流れ
通常、命令は順番に実行されます。しかし、ある変数によって、別のコードを実行したい場合があります。これはコントロールフロー文と呼ばれるもので、プログラムの通常の流れを変更するものです。例えば、マリオが落下している場合、まだ地面に落ちていないかチェックするコードにジャンプします。

オタクな夜2週目。NESアーキテクチャの概要

前週：数の体系とプログラミングのコアとなる考え方

今週：主要な構成要素を網羅したファミコンアーキテクチャの全体像。汎用コンピュータは、コードを格納する場所（ROM）、変数を格納する場所（RAM）、コードを実行するプロセッサ（CPU）がすべて同じように配置されています。ファミコンはさらに、グラフィックスを生成するプロセッサ（PPU）と、オーディオを生成するCPUの一部（APU）が追加されています。ここでのすべては非常に一般的なものであり、次の数週間であなたが望む以上の詳細を持つことになります。

NESのシステムアーキテクチャ

KB - メモリのサイズはキロバイトまたはKBで表示されます。1KB = 1024バイトです。すべて2の累乗なので、1000の代わりに2^10=1024が使われます。大文字と小文字が異なると意味が変わることがあります。KbはKilobits（キロビット）。1バイト＝8ビットなので、Kbを8で割るとKBになります。

ROM - Read Only Memory（リードオンリーメモリ）、変更不可能なデータを保持します。

RAM - Random Access Memory、読み書きが可能なデータを保持する。電源が切れるとチップは消去される。

PRG - プログラムメモリ、ゲームのコード

CHR - キャラクターメモリ、グラフィックのデータ

CPU - Central Processing Unit、メインプロセッサチップ

PPU - Picture Processing Unit、グラフィックチップ

APU - Audio Processing Unit、CPU内部のサウンドチップ



システム概要

NESはカスタム6502ベースのCPUを搭載し、APUとコントローラ処理を1つのチップに、グラフィックを表示するPPUを別のチップに搭載しています。CPU上でコード命令が実行され、APUとPPUにコマンドを送ります。YoboやNEXのようなNOAC（NES On A Chip）クローンは、これらのパーツをすべて1つのチップに載せています。




CPUには変数を保存するための2KBのRAMが、PPUには2画面の背景グラフィックを保持するための2KBのRAMが接続されているだけです。一部のカートでは、Work RAMまたはWRAMと呼ばれる余分なCPU RAMを追加しています。もしカートがセーブされたゲームを保存する必要がある場合、このWRAMは消去されないようにバッテリーが取り付けられています。また、PPU RAMを増設し、4画面分の背景画像を一度に保存できるようにしたカートもあります。これは一般的ではありません。このチュートリアルの残りの部分では、WRAMや4画面RAMは使用しません。

各カートは少なくとも3つのチップを含んでいます。一つはプログラムコード（PRG）、もう一つはキャラクタグラフィック（CHR）、そして最後がロックアウトを保持します。グラフィックチップはROMではなくRAMであることもあり、ゲームコードはPRGのROMチップからCHRのRAMにグラフィックをコピーすることになります。PRGは常にROMチップです。


ロックアウトチップ
NESとカートの内部には、2つのロックアウトチップもある。ロックアウトチップはゲーム機のリセットを制御する。まずNESのロックアウトは0～15のストリームIDを送出する。その数字をもとに複雑な計算をし、その結果を両チップがお互いに送ります。両方のチップは、相手が何を送信することになっているかを知っているので、何か問題が発生したときにお互いに分かります。そうすると、システムは連続的なリセット・ループに入る。これが、汚れたカートで見られる画面の点滅です。

NESのロックアウトチップの4番ピンをカットすると、カートの中にいると勘違いしてしまいます。NESからのIDを待っている状態なので、リセットされることはありません。もし、NESのロックアウトチップを完全に取り除いてしまうと、リセットボタンを制御してしまうため、システムは機能しなくなります。

非正規のゲーム会社が使用したロックアウト解除装置のほとんどは、カートからの大きな電圧スパイクをファミコンのロックアウトに送るものでした。タイミングが合えばファミコンのロックアウトはクラッシュし、システムをリセットすることができなくなる。任天堂は少しずつ、ファミコンの基板にそれらに対する保護機能を追加していきました。次にファミコンを開けたら、基板のリビジョンナンバーをチェックしてみてください。真ん中にNES-CPU-と書いてあり、次に数字が書いてあります。その数字がリビジョンです。05をお持ちの方は初期のものです。07と09はロックアウト保護機能が追加されています。11が最後のバージョンで、最もロックアウト対策が施されています。ロックアウトディフィーターを使用する非正規カートのほとんどは、NES-CPU-11 システムでは動作しません。


CPUの概要
NESのCPUは6502を改良したもので、Apple 2、Atari 2600、C64、その他多くのシステムと同様の8ビットデータプロセッサである。ファミコンが作られた当時は、コンピュータとしては力不足でしたが、ゲームシステムとしては素晴らしいものでした。

CPUは16ビットのアドレスバスを持ち、最大64KBのメモリにアクセスすることができます。2^16=65536、つまり64KBのメモリ空間の中には、2KBのCPU RAM、PPU/APU/コントローラにアクセスするポート、WRAM（カート上の場合）、PRG ROM用の32KBが含まれています。16ビットのアドレスは16進数で書くので、$記号で始まる4桁の数字になる。例えば内蔵RAMは$0000-0800にあります。0800＝2048、つまり2キロバイトである。32KBではゲーム用としてはすぐに小さくなってしまうので、メモリマッパーが使われるようになった。これらのマッパーは、PRGコードやCHRグラフィックの異なるバンクにスワップすることができます。MMC3のようなマッパーでは、PRGは512KBまで、CHRは256KBまで可能でした。マッパーチップを新規に作ればメモリサイズに制限はないが、PRG128KB、CHR64KBが一般的なサイズだった。




PPUの概要
NESのPPUは、すべてのグラフィック表示を行うカスタムチップです。スプライトとカラーパレット用の内部RAMを搭載しています。NESボード上には背景を保持するRAMがあり、実際のグラフィックは全てカートのCHRメモリから取得します。

プログラムはPPU上で実行されるわけではなく、PPUは常に同じ表示順で進みます。色やスクロールなど、一部のオプションを設定するだけです。PPUは一度に1つのTVスキャンラインを処理します。 最初にカートCHRメモリからスプライトをフェッチします。もし、走査線上に8個以上のスプライトがある場合は、残りは無視されます。このため、スーパードッジボールなどのゲームでは、画面上でいろいろなことが起きているときに点滅することがあります。スプライトの後、CHRメモリから背景が取り出されます。すべての走査線が終了すると、グラフィックスが送信されない期間があります。これはVBlankと呼ばれ、グラフィックの更新を行うことができる唯一の時間です。PALはVBlank時間が長いので（テレビのブラウン管が画面の上部に戻るとき）、グラフィックの更新にもっと時間を割くことができます。PALのゲームやデモの中には、このVBlank時間の違いからNTSCシステムで動作しないものがあります。NTSCとPALシステムの両方が256x240ピクセルの解像度を持っていますが、上部と下部8行は、通常、256x224になるNTSCテレビによってカットオフされます。テレビのバリエーションによって、さらに0〜8行が切り取られるので、重要な情報を描く前にボーダーを確保する必要があります。

NTSCは60Hzで動作し、PALは50Hzで動作します。NTSCのゲームをPALシステムで実行すると、このタイミングの違いのために遅くなります。また、サウンドも遅くなります。






グラフィックシステムの概要

タイル
グラフィックはすべて8×8ピクセルのタイルで構成されています。マリオのような大きなキャラクターは、複数の8×8タイルで構成されています。背景もすべてこのタイルでできています。タイル方式は、当時としては高価なメモリをあまり必要としないが、ビットマップ画像や3Dグラフィックスのようなものは実際には不可能であることを意味している。ゲーム内のすべてのタイルを見るには、Tile Molesterをダウンロードして、.NESファイルを開いてください。静止画ではないグラフィックが表示されるまでスクロールダウンしてください。小さなタイルがゲームによって並べられ、大きな画像になっているのがわかると思います。
スプライト
PPUは64個のスプライト（マリオのように画面上を動き回るもの）に対して十分なメモリを持っています。1スキャンラインあたり8個のスプライトしか使用できず、それ以上は無視されます。ゲームによっては、画面上にオブジェクトが多すぎると、ちらつきが発生することがあります。
背景
これは横長のグラフィックで、一度にスクロールします。スプライトは、背景の前に表示することも、後ろに表示することもできます。画面は32x30の背景タイルに十分な大きさで、2画面を保持するのに十分な内部RAMがあります。ゲームがスクロールするとき、背景のグラフィックは画面上でスクロールされる前に画面外で更新される。
パターンテーブル
実際のタイルデータが格納されている場所です。これは、カートのROMまたはRAMのいずれかです。各パターンテーブルは、256のタイルを保持しています。1つのテーブルは背景用、もう1つはスプライト用に使用される。現在画面に表示されているグラフィックは、すべてこのテーブルに入っている必要がある。
属性テーブル
2x2のタイル単位で色情報を設定するテーブルです。つまり、16×16ピクセルの領域には、パレットから4色しか選択できないことになります。
パレット
背景色とスプライトの色情報を保持する2つの領域です。各パレットは16色です。

タイルを画面に表示するには、パターンテーブルと属性テーブルからピクセルのカラーインデックスを取得する。そのインデックスをパレットで検索して実際の色を取得します。




すべてのグラフィックを見るには、FCEUXD SPエミュレータをダウンロードしてください。.NESのゲームを開き、ツールメニューからPPU Viewerを選択します。これで、すべてのアクティブな背景タイル、すべてのアクティブなスプライトタイル、およびカラーパレットが表示されます。次に、ツールメニューから「ネームテーブルビューア」を選びます。これで、画面に表示される背景が表示されます。SMBのようなスクロールするゲームを選ぶと、画面外の背景部分が更新されていくのがわかります。

前週：NESアーキテクチャの概要

今週：6502の詳細とアセンブリ言語の入門に入ります。 asmの使い方とNESの仕様のレッスンは、一緒に分割して行われます。他にも6502のウェブサイトや良い本がたくさんありますので、そちらを参考にしてください。



6502アセンブリ
ビット - コンピュータにおける最小の単位。

バイト - 8ビットを合わせて1バイトとし、0から255までの数値を表します。2つのバイトを合わせると16ビットとなり、0から65535までの数値が形成される。

命令 - プロセッサが実行する1つのコマンド。命令は順番に実行されます。

コードレイアウト
アセンブリ言語では、主に5つのパーツがあります。一部のパーツは、アセンブラが正しく使用できるように、特定の水平位置に配置する必要があります。

命令
命令とは、アセンブラに送るコマンドのことで、メモリ内のコードを探すなどの処理を行います。.で始まり、字下げされます。タブや4つのスペースを使う人もいますが、私は2つのスペースを使っています。このサンプルでは、ゲームROMの領域である$8000から始まるコードをアセンブラに指示しています。
.org $8000

ラベル
ラベルは左端に配置され、末尾に:が付きます。ラベルは、コードを整理し、読みやすくするために使用するものです。アセンブラはラベルをアドレスに変換します。ラベルのサンプル
.org $8000
MyFunction:
アセンブラが実行されると、MyFunctionを$8000に設定するためにfind/replaceが実行されま す。もし、MyFunctionを使用するコードがあれば、次のようになります。
 STA MyFunction
このようにMyFunctionを使うコードがあれば、find/replaceが行われます。
 STA $8000

オペコード
オペコードは、プロセッサが実行する命令で、ディレクティブと同様にインデントされています。 このサンプルでは、JMPが、プロセッサにMyFunctionラベルにジャンプするよう指示するオペコードです。
  .org $8000
MyFunction:
 JMP MyFunction

オペランド
オペランドは、オペコードの追加情報です。オペコードは、1つから3つのオペランドを持ちます。この例では、#$FF がオペランドです。
  .org $8000
MyFunction:
 LDA #$FF
 JMP MyFunction

コメント
コメントは、コードが何を行っているかを英語で理解するためのものです。コードを書いていて、後で戻ってきたとき、コメントがあると助かります。すべての行にコメントが必要なわけではありませんが、何が起こっているかを説明するのに十分な数であるべきです。コメントは ; で始まり、アセンブラでは完全に無視されます。コメントは水平方向ならどこにでも書くことができますが、通常は長い行を越えて間隔をあけて書くことになります。
.org $8000
MyFunction: ; loads FF into accumulator
 LDA #$FF
 JMP MyFunction
このコードはループを継続的に実行し、その度に16進数の値$FFをアキュムレータ にロードします。



6502 プロセッサの概要
6502 は、16 ビットのアドレスバスを持つ 8 ビットプロセッサです。バンクスイッチなしで64KBのメモリにアクセスすることができます。NESでは、このメモリ空間を、RAM、PPU/オーディオ/コントローラ・アクセス、ゲームROMに分割しています。
$0000-0800 - 内蔵RAM、NESの2KBチップ
$2000-2007 - PPUアクセスポート
$4000-4017 - オーディオとコントローラアクセスポート
$6000-7FFF - ゲームカート内のオプションWRAM
$8000-FFFF - ゲームカートROM
ゲームカート部のいずれかをバンクスイッチしてより多くのメモリにアクセスできますが、メモリマッパはこのチュートリアルに含まれていないでしょう。



6502アセンブリの概要
6502のアセンブリ言語は、3文字の命令コード "opcode "から始まります。56個の命令がありますが、そのうち10個は頻繁に使用する命令です。多くの命令では、オペコードの後に10進数か16進数で値を書きます。その値が#で始まっていれば、実際の数字を使うことを意味します。もしその値が#で始まっていなければ、そのアドレスの値を使うということです。つまり、LDA #$05は値5をロードすることを意味し、LDA $0005はアドレス$0005に格納されている値をロードすることを意味します。



6502 レジスタ
レジスタは、値を保持するプロセッサ内部の場所です。6502には、3つの8ビット・レジスタと1つのステータス・レジスタがあり、これらを使用します。このチュートリアルでは説明しないレジスタもあります。

アキュムレータ
アキュムレータ (A) は、データのロード、ストア、比較、演算を行うための主要な 8 ビットレジスタです。最も頻繁に使用する操作は次のとおりです。
LDA #$FF ;16進数の値$FF（10進数の256）をAにロード
STA $0000 ;アキュムレータをメモリ位置$0000（内蔵RAM）にストア

インデックス・レジスタX
インデックス・レジスタX（X）も8ビット・レジスタで、通常はカウントやメモリ・アクセスに使用します。 ループでは、Aを使ってデータを処理しながら、このレジスタを使ってループが何回経ったかを記録します。頻繁に行われる操作としては
LDX $0000 ;メモリ位置$0000の値をXにロード
INX ;Xをインクリメント X = X + 1

インデックスレジスタY
インデックスレジスタY（Y）は、Xとほとんど同じように動作します。いくつかの命令（ここではカバーしません）は、Xでのみ動作し、Yでは動作しません。いくつかの操作は次のとおりです。 
STY $00BA ;store Y into memory location $00BA

このコードは16ビット数まで拡張できるが、比較は難しくなる。その代わり、より複雑なループとテーブルを使ったシフトのシリーズが使われる。このコードでは、2進数の値をキャリービットにシフトして、最終的な10進数の結果に数値を加算するタイミングを指示しています。このコードは私が書いたものではなく、徳丸さんの投稿（http://nesdev.parodius.com/bbs/vi）から引用したものです。このフォーラムスレッドには、他にもさまざまな変換スタイルの例があります。

数字を表示する
数字を10進数で表示したら、それを画面に表示する必要があります。上のコードでは、すべての結果が 00000 = $00 $00 $00 $00 となります。もし、背景の数字タイルが0から始まるなら、これでうまくいきます。しかし、ASCIIを使用している場合は、各桁にオフセットを追加する必要があります。0という数字のASCIIコードは$30なので、背景に書き込む前に各桁に$30を追加すればいいのです。もし、あなたのコードが最初の桁の比較/ラップ方法を使用している場合、$3Aと比較して$30にラップすることで自動的に処理することができます。この場合、数字を00000にクリアするときに、各桁を$00ではなく$30に設定することを確認する必要があります。背景のタイルの位置は自分で決められるので、桁のタイルのオフセットは好きなように設定できます。

すべてをまとめる
pong2.zipのサンプルファイルをダウンロードして解凍してください。ゲームプレイの状態とボールの動きのコードは、pong2.asm ファイルにあります。そのファイルと mario.chr と pong2.bat が NESASM3 と同じフォルダにあることを確認し、 pong1.bat をダブルクリックします。これでNESASM3が実行され、pong2.nesが生成されるはずです。そのNESファイルをFCEUXD SPで実行すると、スコアが表示されます!今は、ボールが側壁でバウンドするたびにスコアが加算されるだけです。

スコア変数を2つ作って、両方描画してみてください。また、他の2進数-10進数コンバータを使用して、毎回スコアに1以上を加算することもできます。また、DrawScoreでは、得点の桁をチェックして、先頭のゼロを描かないようにすることができます。その代わり、背景に描画するときは空白に置き換えます。
